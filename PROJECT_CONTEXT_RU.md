# SpaceshipCrew — краткий контекст проекта

Короткая выжимка для подгрузки в сессии реализации задач по промтам из `MVP_TASKS_AND_PROMPTS_RU.md`. Подробности и числа — в `GAME_DESIGN_DOCUMENT.md` и `DEMO_MVP_IMPLEMENTATION_PLAN.md`.

## Что это за проект

**SpaceshipCrew** — кооперативный sci-fi экшен-выживание: игроки — экипаж одного корабля, роли взаимозависимы, давление создаётся **ресурсами и состоянием корабля** (топливо, кислород, поломки, бой), а не таймерами миссий. Камера **от третьего лица**, свободное перемещение по отсекам. Любой игрок может временно занять любую станцию; эффективность зависит от специализации. Размер партии гибкий: больше людей — выше потребление кислорода (сигнал в конструкторе, не жёсткий кап).

## Технологии и репозиторий

- **Движок:** Unreal Engine 5.7 (ПК).
- **Важно для реализации:** при разработке ориентируемся на API/поведение именно UE 5.7 (включая Editor/Slate/Viewport), чтобы избегать несовместимых решений из других версий.
- **Базовая ветка MVP:** `feature/demo-mvp-implementation`.
- **Задачи:** отдельные ветки `task/<ID>-<short-name>`; по завершении — **PIE-ready** (запуск без blocker/crash, новый функционал проверяем, минимум тестов по правилам промта).

## Документирование кода (C++)

- Для **классов и методов** добавляй описания **на русском языке** и в **заголовочных** (`.h`), и в **файлах реализации** (`.cpp`): в `.h` — назначение, контракт, важные предусловия/побочные эффекты; в `.cpp` при необходимости — уточнения по шагам алгоритма или нетривиальным веткам. Единый стиль комментариев (например `//` или Doxygen `/** */`) выдерживай так же, как в соседнем коде модуля.

## Цель Demo MVP

Замкнуть демонстрируемый цикл: **главное меню → конструктор/тренировки/новая игра → стартовая станция → вылет → возврат → сдача контракта**, плюс базовая экономика рейса, autosave/rejoin, vote-kick baseline. Детальный порядок — в `DEMO_MVP_IMPLEMENTATION_PLAN.md` (issues #1–#12).

## Ключевые системы (на уровне смысла)

| Область | Суть для реализации |
|--------|----------------------|
| **Конструктор корабля** | Цепочка задач **T02a** (данные + editor tooling) → **T02b** (логика сборки, валидация, масса) → **T02c** (сцена + UI). Пресеты — T03. |
| **Роли** | Капитан (маршрут, экономика вылета, координация), пилот, стрелок, инженер, наука и др. по GDD — у каждого свой UI станции. |
| **Экономика MVP** | Награды при **сдаче** на станции, округление до целых, займ с удержаниями, зарплаты/фонды по спеке GDD; расширяемая модель данных. |
| **Боты** | Найм, роли, станции; прокачка дешевле найма «топа» сразу; зарплата от уровня. |
| **Сеть** | Серверная авторитетность, предсказание там, где уместно, rejoin со снапшотом состояния — по разделу GDD про сеть. |
| **Сохранения** | Autosave перед вылетом; при провале кампании — откат к последнему автосейву на станции (правила — в GDD). |

## Ограничения контента

Команды художников нет: примитивы UE, greybox, бесплатные ассеты с фиксацией в `Content/THIRD_PARTY_ASSETS.md` при использовании.

## UI (подход к реализации)

- **Два параллельных базовых слоя** (на выбор экрана, не смешивать в одном виджете без нужды):
  - **Slate:** абстрактный `SSpaceshipMainMenuBase` (`SCompoundWidget`) — общая геометрия, переключение экранов, крупная типографика; конкретный экран наследует и реализует содержимое (например `BuildMainMenuBody()`), сборка через `Construct` в C++.
  - **UMG / Blueprint:** абстрактный `USpaceshipCrewMenuWidgetBase` (`UUserWidget`) — для ручной сборки разметки в редакторе: наследование Blueprint от этого класса, общая логика и начальная разметка при необходимости в `NativeConstruct` в C++.
- **Текущее главное меню MVP** остаётся на Slate (`SSpaceshipMainMenu` от `SSpaceshipMainMenuBase`), чтобы не тянуть UMG-ассеты до отдельной задачи; переход на UMG — через наследника `USpaceshipCrewMenuWidgetBase` и подключение в PlayerController/HUD по мере готовности.
