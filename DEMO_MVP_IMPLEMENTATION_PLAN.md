# План реализации Demo MVP

## Ветка

- Рабочая ветка: `feature/demo-mvp-implementation`
- Цель: получить демонстрируемый цикл `меню -> тренировка -> кампания -> сдача контракта`.

## Базовый скоуп

Реализация опирается на issues:

- `#1` MVP A1: Главное меню и точки входа
- `#2` MVP A2.0: Работоспособный конструктор кораблей (внутренняя декомпозиция по `MVP_TASKS_AND_PROMPTS_RU.md`: **T02a** → **T02b** → **T02c**)
- `#3` MVP A2.1-A2.2: Старт новой кампании (корабль, бюджет, роль)
- `#4` MVP A3: Загрузка стартовой станции
- `#5` MVP B1: Тренировки по ролям
- `#6` MVP B2: Базовые UI интерфейсы станций v1
- `#7` MVP C1: Найм и назначение ботов на станции
- `#8` MVP C2: Demo цикл станция -> вылет -> станция -> сдача
- `#9` MVP C3-C4: Экономика рейса, autosave и rejoin
- `#10` MVP C5: Vote-kick и анти-абьюз baseline
- `#11` MVP D1: Полировка demo пути и стабилизация
- `#12` MVP D2: First-pass баланс экономики и ботов

## Порядок реализации (рекомендуемый)

### Этап 1 - Вход в игру и стартовый поток

1. Реализовать `#1` (каркас главного меню) с 3 ключевыми действиями:
   - Конструктор
   - Тренировки
   - Новая игра
2. Реализовать `#2` (минимальный конструктор корабля):
   - установка/стыковка базовых модулей
   - валидация сборки
   - сохранение пользовательского пресета корабля
3. Реализовать `#3` (старт новой игры):
   - выбор пресета или кастомного корабля
   - проверка бюджета
   - выбор роли (по умолчанию капитан)
4. Реализовать `#4` (загрузка стартовой станции).

Критерии выхода:
- Игрок может запустить игру, собрать/выбрать корабль, выбрать роль и загрузиться на станцию.

### Этап 2 - Тренировки и UI станций

5. Реализовать `#6` (базовые интерфейсы станций v1):
   - навигация капитана
   - режимы/состояние пилота
   - HUD турели
   - панель энергии/ремонта инженера
6. Собрать `#5` (тренировочные миссии, в любом порядке):
   - капитан
   - пилот
   - стрелок
   - инженер

Критерии выхода:
- Все 4 тренировки доступны из меню и проходятся.

### Этап 3 - Ядро кампанийной петли

7. Реализовать `#7` (найм/назначение ботов на станции).
8. Реализовать `#9` (экономика рейса + autosave + rejoin):
   - начисление награды при сдаче миссии
   - расходы, разделение фондов, удержания по займу
   - autosave перед вылетом
   - rejoin со снапшотом инвентаря/состояния
9. Реализовать `#8` (полный demo-цикл):
   - станция -> старт вылета -> полет -> станция назначения -> сдача.

Критерии выхода:
- Полный демо-цикл кампании работает без ручных обходов.

### Этап 4 - Кооп-защита и полировка

10. Реализовать `#10` (vote-kick и baseline анти-абьюз).
11. Провести bugfix/stability проход по `#11`.
12. Провести first-pass баланс по `#12` через Playtest Sheet.

Критерии выхода:
- Demo-путь выполняется 3 раза подряд без blocker/crash.
- Экономика и поведение ботов в целевых KPI-диапазонах.

## Технические рабочие пакеты

- **WP-01 Навигация UI**: маршрутизация меню, переходы экранов.
- **WP-02 Данные конструктора**: схема модулей, совместимость, сохранение пресета.
- **WP-03 Инициализация сессии**: стартовый payload (корабль + роль + бюджет).
- **WP-04 Runtime станции**: интерактивные объекты, вход на станции ролей.
- **WP-05 Runtime тренировок**: короткие изолированные сценарии по ролям.
- **WP-06 Runtime ботов**: найм, назначение, занятость станций.
- **WP-07 Runtime экономики**: реализованный доход, списания, округление, разделение фондов.
- **WP-08 Persistence/сеть**: запись autosave-слотов, восстановление rejoin.
- **WP-09 Кооп-ограничения**: vote-kick, cooldown, guardrails анти-абьюза.
- **WP-10 Стабилизация**: проверка demo-сценария, закрытие критичных багов.

## Чеклист готовности Demo MVP

- [ ] В главном меню работают входы в Конструктор/Тренировки/Новую игру
- [ ] Кастомный корабль можно собрать, провалидировать и сохранить
- [ ] Новая игра корректно проверяет бюджет корабля
- [ ] Выбранная роль применяется при первом спауне
- [ ] Тренировки по 4 ключевым ролям проходят успешно
- [ ] На станции можно нанять ботов и настроить состав
- [ ] Кампанийный вылет и сдача миссии работают end-to-end
- [ ] Экономический результат рейса считается корректно и округляется до целых
- [ ] Autosave + rejoin работают без дублирования данных
- [ ] Vote-kick работает по заданным правилам
- [ ] Demo-путь проходит 3 раза подряд без blocker/crash
